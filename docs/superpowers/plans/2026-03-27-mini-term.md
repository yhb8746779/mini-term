# Mini-Term 实施计划

> 原始完整计划文件在项目初始化时被 create-tauri-app --force 删除。此为任务索引摘要。

**Goal:** 构建基于 Tauri 2 + React + TypeScript 的桌面终端管理工具

**Tech Stack:** Tauri 2, React 18, TypeScript, xterm.js, portable-pty, allotment, notify, Tailwind CSS, Zustand, Vite

**Spec:** `docs/superpowers/specs/2026-03-27-mini-term-design.md`

**重要约定:** Rust 端所有对外结构体必须添加 `#[serde(rename_all = "camelCase")]`

---

## 任务列表

### Task 1: 项目脚手架与依赖安装 ✅
- 创建 Tauri 2 + React + TS 项目
- 安装前端依赖（xterm.js, allotment, zustand, tailwindcss）
- 添加 Rust 依赖（portable-pty, notify, ignore, dirs, windows）

### Task 2: TypeScript 类型定义与 Zustand Store ✅
- `src/types.ts` — 所有接口定义
- `src/store.ts` — Zustand 全局状态管理
- `src/hooks/useTauriEvent.ts` — Tauri 事件监听 hook

### Task 3: Rust 配置模块 ✅
- `src-tauri/src/config.rs` — load_config / save_config 命令

### Task 4: Rust PTY 管理器 ✅
- `src-tauri/src/pty.rs` — create_pty / write_pty / resize_pty / kill_pty
- Channel + 16ms flush 定时器批量缓冲

### Task 5: 应用外壳布局 ✅
- `src/App.tsx` — 三栏 allotment 布局
- `src/components/StatusDot.tsx` — 状态指示圆点

### Task 6: 单终端实例（xterm.js 集成） ✅
- `src/components/TerminalInstance.tsx` — xterm.js + PTY 通信 + WebGL + 拖拽接收

### Task 7: Tab 栏与 Tab 管理 ✅
- `src/components/TabBar.tsx` — Tab 栏组件
- `src/components/TerminalArea.tsx` — 终端区域容器

### Task 8: 递归分屏布局 ✅
- `src/components/SplitLayout.tsx` — allotment 递归分屏

### Task 9: 项目列表面板 ✅
- `src/components/ProjectList.tsx` — 项目添加/删除/切换

### Task 10: Rust 文件系统操作 ✅
- `src-tauri/src/fs.rs` — list_directory / watch_directory / unwatch_directory

### Task 11: 文件树组件 ✅
- `src/components/FileTree.tsx` — 懒加载树形文件浏览

### Task 12: Rust 进程状态检测 ✅
- `src-tauri/src/process_monitor.rs` — Windows 进程树检测

### Task 13: Rust AI 会话历史读取 ✅
- `src-tauri/src/ai_sessions.rs` — AISessionReader + Claude session 解析

### Task 14: AI 会话历史面板 ✅
- `src/components/AIHistoryPanel.tsx` — 按项目展示历史 session

### Task 15: 集成完善与收尾 ✅
- 右键分屏菜单、PTY 清理、退出处理

## 依赖关系

```
Task 1 (脚手架)
  → Task 2 (类型/状态)
  → Task 3 (配置) → Task 9 (项目列表)
  → Task 4 (PTY) → Task 6 (终端) → Task 7 (Tab) → Task 8 (分屏)
                 → Task 12 (进程检测)
  → Task 5 (布局)
  → Task 10 (文件系统) → Task 11 (文件树)
  → Task 13 (AI 读取) → Task 14 (AI 面板)
  → Task 15 (集成收尾)
```
